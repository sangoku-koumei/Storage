# 自律型エージェントの技術的ブレイクスルー (2025年)

## 1. 「Agentic AI」へのシフト (能動的 vs 受動的)
*   **概念**: *Generative AI* (プロンプトに応答する) から *Agentic AI* (目標を追求する) への移行。
*   **能力**: エージェントは長期的なタスクを計画し、サブステップに分解し、実行し、結果を観察し、失敗した場合は自己修正することができる。
*   **自律性**: 「人間がループ内にいる（Human-in-the-loop）」状態から、「人間はループの上にいる（Human-on-the-loop）」、そして定型タスクについては事実上「ループの外（out of the loop）」で機能する状態へとシフトしている。

## 2. エージェント間相互運用プロトコル
*   **解決された課題**: 以前は、SalesforceのエージェントはGoogle Workspaceのエージェントと簡単に会話できなかった。
*   **ブレイクスルー**: **Model Context Protocol (MCP)** やその他（Google ADK, Cisco SLIM）のプロトコルの登場により、「エージェントのウェブ」が構築された。
*   **インパクト**: エージェント同士がタスクを引き継ぐことが可能に。「マネージャーエージェント」が、「コーダーエージェント」にコーディングタスクを、異なるプラットフォーム上の「DevOpsエージェント」にデプロイタスクを委任できる。

## 3. Large Action Models (LAMs) と Computer Use
*   **技術**: テキストだけでなく、*行動*（GUI操作、APIコール）について訓練されたモデル。
*   **結果**: コンピュータの画面を「見て」、マウスやキーボードを仮想的に使用できるエージェント。単にテキストを生成するだけでなく、ボタンをクリックし、フォームに入力し、APIのない複雑なソフトウェアインターフェースを操作する。

## 4. 協調的知能 (マルチエージェントシステム)
*   **構造**: 特化型エージェントのオーケストレーション（群知能）。
*   **メリット**: 1つの巨大なモデルがすべてを行おうとするのではなく、小規模で高度に専門化されたエージェントが協力する。1つが計画し、1つが批評し、1つが実行する。
*   **メモリと状態**: 高度な永続メモリにより、エージェントは現在のセッションだけでなく、数ヶ月にわたってユーザーの好みや過去のプロジェクトの文脈を記憶できる。

## 5. 信頼とガバナンスのレイヤー
*   **ガードレール**: エージェントレベルでの「Constitutional AI（憲法的AI）」の実装。ユーザーのプロンプトに関わらず、エージェントが*やってはいけないこと*（例：本番データベースの削除、金融詐欺への加担）についての埋め込みルールを持つ。
