# Auto Accept & Git Save Tool

画面上に出てくる「Accept（承認）」ボタンなどを自動でクリックし、さらに定期的にGitへの自動バックアップを行うツールです。

## 昨日
1.  **自動クリック**: 事前に保存した画像（複数可）と一致するボタンが画面に出ると、自動でクリックします。
2.  **Git自動保存**: 15分ごとに `git add .` -> `git commit` を自動実行します。（設定でPushも可）

## セットアップ

### 1. 必要なライブラリの確認
既にインストール済みのはずですが、エラーが出る場合は以下を実行してください。
```powershell
pip install pyautogui pillow
```
※ `opencv-python` は今回使用しない方式に変更したため不要ですが、精度を上げたい場合は入れても構いません。

### 2. 画像の用意（重要！）
このツールは**フォルダ内のすべての `.png` ファイル**を「クリック対象」として認識します。

1.  クリックしたい「Accept」ボタンが表示されたら、スクリーンショットを撮ります。
    *   **Windows**: `Win + Shift + S` で範囲指定してクリップボードへ → ペイントなどに貼り付けて保存。
    *   **ポイント**: ボタンの**絵柄部分だけ**をギリギリで切り取ってください。余白が多いと認識しません。
2.  保存した画像をこのフォルダ `00\AutoAccept` に入れます。
3.  画像の種類が複数ある場合（例：Acceptボタン、OKボタン、Yesボタンなど）は、全てこのフォルダに入れてください。名前はなんでもOKです（例 `pattern1.png`, `pattern2.png`）。

### 3. Gitの設定（任意）
スクリプト内の `DO_PUSH = False` を `True` に書き換えると、コミット後に自動で `git push` まで行います。
※ リモートリポジトリの設定が済んでいる必要があります。

## 使い方

ターミナルで以下を実行します。

```powershell
python auto_accept.py
```

*   **停止方法**: ターミナルで `Ctrl + C` を押すと止まります。
*   **ログ**: ボタンを見つけてクリックした時と、Git保存が行われた時にログが出ます。

## 【深層解説】自動承認（AutoAccept）ツールの技術的制約と「自動化の哲学」

### 1. 「画像認識 vs 座標指定」のトレードオフ
このツールは `pyautogui` による画像認識（テンプレートマッチング）を採用しています。座標指定（クリックする場所を固定する）に比べて柔軟性が高い一方で、「ボタンの色が変わる」「フォントのレンダリングが変わる」「ダークモードに切り替わる」といった変化に極端に弱いです。
- **ヒント**: 認識精度を上げるには、ボタンの「テキスト部分」ではなく、ボタンの「カド（角）」や独特な「アイコン」を含めてキャプチャするのがコツです。

### 2. 人間らしさ（Human-like Behavior）の実装
現在のコードは検知した瞬間にクリックしますが、これはいわゆる「ボット検知」に引っかかる可能性があります。将来的には、クリック前に「ランダムなミリ秒の待機」や「マウスカーソルの滑らかな移動（ベジェ曲線）」を実装することで、より人間に近い挙動をシミュレートできます。

### 3. 「Git自動保存」がもたらす究極の安心感
本ツールの真の価値は、クリック自動化よりも「15分ごとの強制Gitコミット」にあります。
- **なぜ必要か？**: 思考に没頭していると、バックアップを忘れ、最悪の場合「1日分の作業が消える」事故が起きます。
- **哲学**: 完璧なコミットメッセージよりも「一歩前の自分に確実に戻れる安心感」を重視しています。メッセージが `auto save 2024-XX-XX` であっても、履歴があること自体が開発者の精神衛生を保ちます。

### 4. セキュリティと倫理の境界線
このツールは「あなたの代わりにYESを押す」ものです。これは非常に強力ですが危険でもあります。
- **警告**: 金銭的な承認ボタン、重要なデータ削除の確認ボタンがある画面では、絶対にこのツールを動かさないでください。ツールは「善悪」を判断しません。
