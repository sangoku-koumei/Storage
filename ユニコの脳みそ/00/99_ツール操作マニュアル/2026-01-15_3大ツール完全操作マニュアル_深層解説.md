    *   **デザイン**: シンプルで清潔感のある白ベース（ユニコーン企業のドキュメント風）。
    *   **販売単価**: 5万〜10万円

### 🅱️ Manga Mode (Comic Style) 🦸
*   **用途**: 新人教育、外国人スタッフ向け、コンプライアンス研修。
*   **特徴**:

    *   **Cast Selector**: 「標準」「工場」「営業」の3パターンからキャラ選定可能。
    *   **UX**: 縦スクロール型（ウェブトゥーン形式）で、スマホでの閲覧に最適化。
    *   **販売単価**: 20万〜30万円（「わかりやすさ」への対価）

### 🅾️ Visual Mode (Dashboard) 📊

## 1-3. [Feature] ヒアリングボット (Hearing Bot) の攻略

「何を書けばいいかわからない」というクライアントのために、AIがインタビュアーになる機能。

### 操作フロー
1.  **メモ入力**: 「新人研修の資料を作りたい。挨拶と名刺交換について」程度のメモを入力。
2.  **ボタンクリック**: 「🤖 ヒアリング開始」を押す。
3.  **AIからの質問**:
    *   Q1: 「新人のターゲット層は？（高卒？大卒？）」
    *   Q2: 「特に強調したいNG行動はありますか？」
    *   Q3: 「社風は体育会系ですか？ フラットですか？」
4.  **回答**: これに答えるだけで、AIは「文脈」を理解し、構成案の精度が劇的に向上する。

### Pro Tip
*   **「わざと空欄」**: 全て答える必要はない。分からない場合は空欄で進めば、AIが一般的なベストプラクティス（標準解）を埋め込んでくれる。

## 1-4. [Advanced] マンガ＆キャスト設定 (Cast Selector)

v4.0の目玉機能。クライアントの業種に合わせて「演者」を変える。

### キャストの種類
1.  **Standard（標準）**:
    *   **Rookie**: マナブ（22歳）。少しだらしないが愛嬌のある若手。
    *   **Mentor**: シゴト先輩（28歳）。メガネの女性。冷静沈着。
    *   **適用**: 一般的なオフィスワーク、IT企業、事務職。
2.  **Factory（工場・現場）**:
    *   **Rookie**: ゲンバ君（20歳）。ヘルメット着用、安全靴。
    *   **Mentor**: オヤジさん（55歳）。首にタオル、無精髭だが目は優しい。
    *   **適用**: 建設業、物流倉庫、製造ライン、運送業。
3.  **Sales（営業・体育会）**:
    *   **Rookie**: セールス君（24歳）。ジェルで固めた髪、スリムスーツ。
    *   **Mentor**: マネージャー（35歳）。3ピーススーツ、オールバック。
    *   **適用**: 不動産営業、保険代理店、光通信系。

### 技術的裏側 (Character Sheet Anchoring)
選択されたキャスト情報は、`input_text` とは別に、画像生成プロンプトの「System Prompt」としてDALL-E 3に注入される。
```python
# 実際の注入プロンプト例
Prompt += """
[Character Reference]
Name: ゲンバ君
Appearance: Young male, yellow safety helmet, green work uniform...
"""
```
これにより、コマが変わっても「同じヘルメットの同じ顔」が生成され続ける。

# 第2部：⚡ AI Sales Automation Tool v5.0 (完全解説)
**〜無人営業マシン〜**

「寝ている間にアポを取る」を実現するシステム。
v5.0（Pro Hardened）では、GoogleやプロバイダのBAN回避機能が強化されている。

## 2-1. [Agent Mode] 自律型リスト収集

### DuckDuckGo Search Integration
*   **機能**: 「渋谷 マーケティング会社」などのキーワードを入れると、検索結果から企業の公式URLだけを抽出する。
*   **フィルター**: Wikipedia、ポータルサイト（iタウンページ等）、求人サイトを除外し、「本物の企業サイト」だけをリスト化するロジックが組み込まれている。

### Anti-Ban Scraping (Selenium Fallback)
*   **仕組み**:
    1.  まず高速な `requests` (HTTP) でアクセスを試みる。
    2.  403 Forbidden（ブロック）が返ってきた場合、自動的に重量級の `Selenium` (Headless Browser) に切り替える。
    3.  人間のようなマウスの動きを模倣し、情報を取得する。
*   **取得情報**:
    *   会社概要（Vision, Mission）→ 個別メッセージ生成に使用。
    *   お問い合わせフォームURL。
    *   代表メールアドレス（あれば）。

## 2-2. [Check] ファクトチェック＆評判保護

### Fact-Check Engine
*   **機能**: AIが生成したメール文面（特に「御社の〇〇事業に感銘を受け...」の部分）が、スクレイピングした事実と合致しているかを再検証する。
*   **効果**: 「そんな事業やってないよ」という致命的なミスを防ぎ、返信率を高める。

### Reputation Guardian
*   **機能**: 1日の送信上限（Daily Limit）を強制的に守らせる。
*   **設定値**: デフォルトは「1日 50通」。これを超えると、エラーが出て送信がストップする。
*   **理由**: GmailやOutlookのスパムフィルターは「急激な送信数増加」を検知してドメインをBANする。これを防ぐための安全装置。

## 2-3. [CRM] 顧客管理ダッシュボード

### ステータス管理
SQLiteデータベースで以下のステータスを管理する。
*   `Pending`: リスト入りしたが未送信。
*   `Generated`: 文面作成済み。
*   `Sent`: 送信済み。
*   `Replied`: 返信あり（手動更新）。
*   `Blacklist`: 送信禁止（クレーム等）。

### バッチ処理 (CSV Import)
*   **機能**: 手持ちのリスト（Excelなど）がある場合、CSVとしてアップロードすれば一括でDBに取り込める。
*   **フォーマット**: `Company Name`, `URL` の2列があればOK。

# 第4部：3大ツール連携による「ゴールデン・ワークフロー」

これら3つのツールを個別に使うのではなく、**連携（Pipeline）** させることで最強のエージェンシー業務が可能になる。

### Step 1: ターゲット選定 (Research Tool)
1.  **Research Tool** で「伸びている業界」をリサーチする（例：サウナ業界）。
2.  その業界の課題（例：予約管理がアナログ）を特定する。

### Step 2: 営業爆撃 (Sales Tool)
1.  **Sales Tool** に「サウナ 東京」と入力し、100社のリストを自動生成する。
2.  リストに対して、「予約自動化システム」の提案メールを一斉送信する。
3.  メール内には、**Manual Tool (LP Mode)** で作った「サウナDX導入サービスのLP」へのリンクを貼っておく。

### Step 3: 受注＆納品 (Manual Tool)
1.  興味を持った施設から返信が来る。
2.  商談で、**Manual Tool (Visual Mode/Manga Mode)** を使い、従業員向けのマニュアル作成を受注する（単価30万）。
3.  受注したら、ツールを使って爆速でマニュアルを作成・納品する。
4.  余った時間で次のターゲット（例：ジム業界）をリサーチする。
