# 潜在意識診断_全52問マスターデータ：サイト実装用ロジック定義書

> **【ドキュメント概要】**
> 本書は、Web診断システムに実装するための「全質問・全選択肢・配点ロジック」を定義したマスターデータである。
> エンジニアはこの値をそのままデータベースに格納することで、意図通りのプロファイリングが可能となる。

---

## 1. 診断のカテゴリ構成

全52問は以下のカテゴリに分類される。

| ID | カテゴリ名 | 測定パラメータ | 問数 |
| :--- | :--- | :--- | :--- |
| **Q01-Q13** | **Part 1: 距離感のOS** | A軸（親密さスタンス） | 13 |
| **Q14-Q26** | **Part 2: 表現のOS** | B軸（本音表現スタイル） | 13 |
| **Q27-Q39** | **Part 3: 脚本のOS** | C軸（ストーリー感覚） | 13 |
| **Q40-Q52** | **Part 4: 制御のOS** | D軸（コントロール欲求） | 13 |

※サブ指標（SV, AC, CO, BD）は、各カテゴリ内に隠し要素として分散配置されている。

---

## 2. 配点ルール（共通）

各質問には4つの選択肢があり、それぞれに以下のパラメーターポイントが付与される。
数値は「強度」を表す（最大5pt）。

*   **Axis Point**: その軸（A+/A-など）への加点。
*   **Sub Point**: サブ指標（SV/AC/CO/BD）への加点（該当する場合のみ）。

## 7. 診断結果ロジック（タイプ判定アルゴリズム）

各軸の合計点（Max 65pt）に基づき、High/Lowを判定する。
※閾値は全回答データの平均値から動的に設定することを推奨（β版では「32pt」を閾値とする）。

*   **A軸スコア** >= 33 → **A+ (接近)** / < 33 → **A- (距離)**
*   **B軸スコア** >= 33 → **B+ (開示)** / < 33 → **B- (内包)**
*   **C軸スコア** >= 33 → **C+ (物語)** / < 33 → **C- (現実)**
*   **D軸スコア** >= 33 → **D+ (制御)** / < 33 → **D- (受容)**

この4つの組み合わせにより、16タイプID（01〜16）を出力する。
