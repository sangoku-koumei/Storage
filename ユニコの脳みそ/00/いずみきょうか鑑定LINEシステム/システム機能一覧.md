## 📊 スプレッドシート構成（27シート）

### メインシート（常時使用）

```
1. users - ユーザー情報
2. applications - 申込み記録
3. deadlines - 締切管理

5. readings - 鑑定結果（3ヶ月分のみ、以降アーカイブ）
6. email_templates - メールテンプレート
7. send_queue - 送信キュー（1ヶ月分のみ、以降アーカイブ）
8. consult_requests - 相談候補
9. consult_decisions - 相談決定
10. appointments - 予約管理
11. products - 商品マスター
12. payments - 決済記録
13. manual_payments - 手動決済記録
14. subscriptions - サブスク管理⭐
15. monthly_fortunes - 月次運勢記録（3ヶ月分のみ）⭐
16. monthly_fortune_schedule - 配信スケジュール⭐
17. monthly_fortune_log - 配信ログ⭐
18. ai_prompts - プロンプト管理
19. ai_settings - AI設定
20. ai_reading_schedule - AI実行スケジュール
21. form_tokens - トークン管理
22. links - URLリンク
23. config - システム設定
24. ops_tickets - 運営チケット
25. kpi_daily - 日次KPI
26. consents - 同意記録
27. logs_2025-11 - ログ（月次でアーカイブ）
```

### アーカイブシート（自動作成）

```
archive_logs_2025-10 - 過去のログ
archive_send_queue_2025-10 - 過去の送信キュー
archive_readings_2025-08 - 過去の鑑定結果（3ヶ月以上前）
archive_monthly_fortunes_2025-08 - 過去の月次運勢
```

**アーカイブルール**:
- ログ: 毎月1日に前月分をアーカイブ
- 送信キュー: 前月以前の完了分をアーカイブ
- 鑑定結果: 3ヶ月以上前をアーカイブ
- 月次運勢: 3ヶ月以上前をアーカイブ

**→ メインシートは常に軽量！**

## 📈 データ軽量化（アーカイブシステム）⭐

### 自動アーカイブ（毎月1日）

```
対象データ:
1. ログ（前月分）
2. 送信キュー（前月以前の完了分）
3. 鑑定結果（3ヶ月以上前）
4. 月次運勢（3ヶ月以上前）

処理:
- アーカイブシートに移動
- 元のシートから削除
- 運営にメール通知

結果:
→ メインシートは常に軽量
→ 動作が常に高速
→ でも過去データは保持
```

### 手動アーカイブ

Apps Script エディタで `runArchiveNow()` を実行

→ いつでも手動でアーカイブ可能
