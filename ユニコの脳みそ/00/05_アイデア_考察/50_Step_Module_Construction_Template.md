# 50ステップ・モジュール型LINE構築テンプレート（ProLine最適化版）

## 1. 構築の考え方（Core Theory）
従来の「一本道」ではなく、**9つの小シナリオ（モジュール）を組み合わせ、相手の熱量に合わせて「スキップ（ワープ）」**させる設計です。

### 構造：3層構造
- **M1〜M9（モジュール）**: 意味のまとまりごとの小シナリオ。
- **R1〜R9（ユニバーサル・リマインド）**: 各モジュールに必ず1つついている「救済・回収用」ステップ。条件で「送らない（0回）」判定も可能。
- **Fastレーン（スキップ機能）**: 熱い人は「教育」を飛ばして「オファー」へ直行する。

### ProLine Free での実装ロジック
- **進行トリガー**: すべてのアンケート・ボタンは「回答時＝即時次ステップ配信」。
- **熱量判定**:
    - アンケートの回答内容（「早く知りたい」等）
    - 進行スピード（登録から短時間で特定ステップに到達）
- **リマインド制御**: 条件分岐（フィルタ）を使用。「すでに回答/クリック済みの人」には送らない設定にする。

## 3. ステップ構築詳細テンプレート

### ■ M1：入口・信頼構築（誰のためのLINEか）
- M1-1：登録直後・挨拶｜「{悩み}」を一緒にほぐすLINEです＋3分で読める宣言
- M1-2：自己紹介｜「すごい人」ではなく「同じ失敗をした味方」としてのストーリー
- M1-3：構造整理｜なぜ{悩み}が解決しないのか？（知識・行動・環境の図解）
- M1-4：NG行動｜よかれと思ってやりがちな「逆効果」な対処法3選
- **R1（リマインド）**: M1未読者向け｜「忙しくて読めてない方へ、M1の要約だけ送ります」

### ■ M2：ライト教育＆ペース選択（分岐の肝）
- M2-1：未来提示｜1ヶ月後・3ヶ月後にどうなっていたいか（現実的な希望）
- M2-2：プレゼント｜無料チェックリスト配布（即メリット）
- M2-3【重要】：状態アンケ｜「この先の進み方、どれが良いですか？（早め/普通/ゆっくり）」
- **R2（リマインド）**: アンケ未回答者へ｜「どれか1つ押すだけでOKです（ポチっと回答）」
- **🔀 分岐ポイント**: 熱い人（pace_fast 且つ 進行が速い）は M3〜M5 をスキップして M6 又は Fastレーンへ

### ■ M3：ディープ教育1（痛みと共感）
- M3-1：放置リスク｜このまま1年続くとどうなるか（少し厳しめ→救い）
- M3-2：失敗事例｜「あとでやろう」で2年損したAさんの話
- M3-3：成功共通点｜うまくいっている人は「能力」ではなく「選び方」が違う
- **R3（リマインド）**: 未読者へ｜「重たい話だったので、要点3つだけまとめました」

### ■ M4：ディープ教育2（解決の地図）
- M4-1：全体マップ｜解決のための5ステップ（保存版）
- M4-2：最初の一歩｜3分でできる小さなタスク（ハードル極低）
- M4-3【行動】：タスク完了アンケ｜「やりました／まだ」ボタン
- **R4（リマインド）**: 未回答者へ｜「まだでも大丈夫。『まだ』ボタンを用意しました」

### ■ M5：心理ブロック＆診断（自分ごと化）
- M5-1：心理ブロック｜動けないのは意志が弱いからではない（ブレーキの正体）
- M5-2：タイプ診断｜あなたのブレーキはどのタイプ？（A/B/C診断）
- M5-3：まとめ｜ここまでの総括＋「次から解決策（商品）の話をします」
- **R5（リマインド）**: 診断未回答へ｜「直感で『これかな』を1つ押すだけでOKです」
- **🔀 合流ポイント**: スキップした熱い人もここから合流

### ■ M6：オファー導入＆全体像（商品の登場）
- M6-1：導入（橋渡し）｜自力でやるか、サポートを受けるか（選択肢の提示）
- M6-2：商品概要｜{商品名}の全体像を3つのポイントで
- M6-3：開発秘話｜なぜこのサービスを作ったのか（想い）
- M6-4【重要】：LP案内｜「詳しくはこれを見て（見るだけでOK）」
- **R6（リマインド）**: LP未クリック者へ｜「時間がない人向けに、3ポイント要約しました」

### ■ M7：オファー深掘り（検討者向け）
- M7-1：事例詳細｜ビフォーアフターの物語（感情の変化を中心に）
- M7-2：FAQ｜聞きづらい本音Q&A（価格・サポート範囲など）
- M7-3：適合診断｜合う人・合わない人を正直に伝える
- **R7（リマインド）**: 検討中タグの人へ｜「迷っているなら、ここだけ確認してください」

### ■ M8：締切〜募集終了（クロージング）
- M8-1：判断基準｜迷っている人のための「整理する3つの質問」
- M8-2：前日連絡｜明日で募集終了します（事務連絡＋応援）
- M8-3：当日連絡｜ラストコール（これで最後です＋見送る選択も尊重）
- **R8（リマインド）**: 終了直前｜「あと◯時間です（本当に最後）」

### ■ M9：長期ナーチャリング（関係維持）
- M9-1：終了挨拶｜読んでくれた全員へのお礼＋今後の方針
- M9-2：保存版｜長期で役立つ原理原則ノウハウ
- M9-3：日常変化｜{テーマ}が整った後の1日の過ごし方
- M9-4：事例シェア｜最近のクライアントの気付き（ミニコラム）
- M9-5：悩み募集｜今後の配信テーマを決めたいので教えてください
- M9-6：次案内希望｜「次の企画があれば優先的に知りたいですか？」
- **R9（リマインド）**: 休眠復活｜「最近忙しいですか？ 頻度落としましょうか？」

## 4. Cursorへの指示プロンプト（コピペ用）
```markdown
あなたは「教育型LINEマーケティングのプロ」です。
以下の【前提】と【構成】に基づいて、LINE配信の原稿を作成してください。

# 前提
- 商品：{商品名を入れる}
- ターゲット：{ターゲットを入れる}
- 目的：信頼構築から自然なオファーへ繋げる
- トーン：責めない、急かさない、スマホで読みやすい（3〜5スクロール以内）

# 作成するモジュール
**M{数字}：{モジュール名}**
（※上のテンプレートから該当するモジュールの役割をコピペしてください）

# 要件
- 各通のタイトル（【】付き）と本文を作成すること。
- 最後に必ず、そのモジュール専用の「リマインド通（R）」を1通作成すること。
- リマインド通は、「まだ見てない人向けに、要点だけ伝える」やさしい内容にすること。
```

## 5. ProLine設定チェックリスト
- [ ] **シナリオ分割**: M1〜M9（または大きくS1〜S4）でシナリオフォルダを分けるか、ステップ管理しやすいように頭に番号を振る（例：01_M1_挨拶）。
- [ ] **アンケート設定**: すべてのアンケート回答時に「キーワード応答」または「ボタンアクション」で次ステップを即時送信するように設定。
- [ ] **タグ設計**: `pace_fast`, `act_lp_view`, `seg_hot_early` など。
- [ ] **リマインド（Rステップ）の条件**: フィルタ機能を使い、「直前のアンケ未回答」かつ「配信から24時間経過」などで送信。
- [ ] **スキップ設定**: M2終了時にタグがある場合、M3〜M5をスキップする設定を行う。
