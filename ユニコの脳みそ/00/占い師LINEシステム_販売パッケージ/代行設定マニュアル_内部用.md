## 🔧 Day 2: スプレッドシート構築（詳細手順）

### Step 1: マスタースプレッドシートをコピー（5分）

```
1. マスタースプレッドシートを開く
   https://docs.google.com/spreadsheets/d/MASTER_ID/

2. 「ファイル」→「コピーを作成」

3. 名前を変更:
   「【顧客名】占い師LINEシステム_202X-XX-XX」

4. 購入者のGoogleアカウントに共有
   権限: 編集者
```

### Step 2: configシート設定（10分）

configシートを開き、以下を設定：

```
sender_name: 購入者の占い師名
sender_email: 購入者のシステム用メール
ops_email: 購入者の運営用メール
openai_api_key: 購入者のOpenAI APIキー
paypal_client_id: （不要・削除）
timezone: Asia/Tokyo
preview_mode: TRUE（初期はプレビュー）
freeze_all: FALSE
freeze_sending: FALSE
```

### Step 3: ai_promptsシート カスタマイズ（30分）

#### SYS_001（システムプロンプト）のカスタマイズ

購入者の占いスタイルに合わせて調整：

**優しい・寄り添う系の場合:**
```
あなたは「[占い師名]」という名前の、相談者に寄り添う占い師です。

【あなたの特徴】
- 温かく包み込むような文体
- 相談者の気持ちに共感する
- 優しく励ます表現
- 希望を持てる言葉選び
...
```

**スピリチュアル・神秘的の場合:**
```
あなたは「[占い師名]」という名前の、高次の存在と繋がる占い師です。

【あなたの特徴】
- 神秘的で深遠な文体
- 魂のメッセージを伝える
- 宇宙の法則を説く
- スピリチュアルな表現
...
```

**現実的・実践的の場合:**
```
あなたは「[占い師名]」という名前の、実践的なアドバイスを提供する占い師です。

【あなたの特徴】
- 明確で具体的な文体
- 実行可能なアドバイス
- 現実的な解決策を提示
- ステップバイステップの指針
...
```

#### PROD_001〜006のカスタマイズ

購入者が選択した鑑定内容に合わせて調整。
不要な商品は削除、追加商品があれば追加。

### Step 4: productsシート設定（15分）

購入者の希望価格に変更：

```
PROD_001: 有料鑑定（基本） → 希望価格に変更
PROD_002: 有料鑑定（詳細） → 希望価格に変更
PROD_005: 恋愛専門鑑定 → 希望価格 or 削除
PROD_006: 仕事・転職鑑定 → 希望価格 or 削除
PROD_007: 月次詳細（単発） → 希望価格 or 削除
PROD_SUB_001: 月次サブスク → 希望価格 or 削除
```

### Step 5: email_templatesシート カスタマイズ（20分）

すべてのメールテンプレートの差出人名を変更：

```
[占い師名] → 購入者の占い師名

例:
「いずみきょうか」→「星読みマリア」
```

メールの文体も占いスタイルに合わせて調整。

### Step 6: linksシート設定（10分）

```
L9101〜L9106: PayPal.Me URLを変更
例: https://paypal.me/購入者ID/金額

L9001〜L9003: Googleフォーム URL（後で更新）
```

### Step 7: Code.gs デプロイ（15分）

```
1. 「拡張機能」→「Apps Script」

2. Code.gsを確認（すでにコピーされている）

3. 「デプロイ」→「新しいデプロイ」
   - 種類: ウェブアプリ
   - 実行ユーザー: 自分
   - アクセス: 全員

4. デプロイURLをコピー（linksシートに記録）

5. 「実行」→「setupSheets」
   ↓
   初回は権限承認が必要
   ↓
   「安全ではない」→「詳細」→「プロジェクト名に移動」
   ↓
   「許可」

6. 実行ログで「setupSheets: スプレッドシート初期化完了」確認

7. 「実行」→「setupTriggers」
   ↓
   トリガーが設定されたことを確認
```

### Step 8: 動作テスト（15分）

```
1. configシートでpreview_mode = TRUE確認

2. 手動でtick()を実行
   ↓
   エラーがないことを確認

3. logsシートでログ確認

4. send_queueが空であることを確認

5. usersシートにテストユーザーを1件追加
   ↓
   自動でuser_idが生成されることを確認
```

## 📝 Day 3: Googleフォーム作成（詳細手順）

### フォーム1: 無料鑑定申込みフォーム（20分）

```
1. テンプレートをコピー
   https://docs.google.com/forms/d/TEMPLATE_FREE/copy

2. タイトル変更:
   「[占い師名]の無料鑑定 お申込みフォーム」

3. 説明文変更:
   占い師名、サービス内容を更新

4. 質問項目確認:
   □ お名前
   □ メールアドレス
   □ 生年月日
   □ ご相談内容

5. 回答先を購入者のスプレッドシートに変更:
   「回答」→「回答先を選択」→「既存のスプレッドシート」
   → 購入者のスプシを選択

6. 送信後のメッセージ変更:
   「[占い師名]が心を込めて鑑定いたします」

7. フォームURLをコピー
   → linksシートのL9002に貼り付け

8. スクリプトエディタで連携設定:
   「...」→「スクリプトエディタ」
   ↓
   以下のコードを追加:
```

```javascript
function onFormSubmit(e) {
  const ss = SpreadsheetApp.openById('【購入者のスプシID】');
  const scriptUrl = ss.getSheetByName('config').getRange('B10').getValue();
  
  const payload = {
    type: 'free_reading_form',
    name: e.values[1],
    email: e.values[2],
    birth_date: e.values[3],
    consultation: e.values[4]
  };
  
  UrlFetchApp.fetch(scriptUrl, {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify(payload)
  });
}
```

```
9. トリガー設定:
   「トリガー」→「トリガーを追加」
   - 関数: onFormSubmit
   - イベント: フォーム送信時

10. テスト送信
    ↓
    スプレッドシートに反映されることを確認
```

## 📱 Day 4: LINE・プロライン設定（詳細手順）

### LINE公式アカウント設定（30分）

#### 既存アカウントがある場合

```
1. 購入者から管理者権限を付与してもらう

2. LINE Official Account Manager にログイン
   https://manager.line.biz/

3. アカウント確認
   - アカウント名
   - アカウントID
```

#### 新規作成の場合

```
1. LINE Official Account Manager にアクセス
   https://manager.line.biz/

2. 「アカウントを作成」

3. 情報入力:
   - アカウント名: 購入者の占い師名
   - 業種: 美容・健康・占い
   - プラン: フリープラン

4. アカウント作成完了
   ↓
   アカウントIDをlinksシートに記録

5. 購入者に管理者権限を付与
```

## 🔧 トラブルシューティング（内部用）

### よくある問題と対処

#### 問題1: Code.gsの権限エラー

```
原因: Googleの権限承認が必要

対処:
1. 「安全ではない」→「詳細」
2. 「プロジェクト名に移動」
3. 「許可」をクリック
```

#### 問題2: フォーム送信が反映されない

```
原因: トリガー設定が未完了

対処:
1. フォームのスクリプトエディタを開く
2. 「トリガー」で onFormSubmit が設定されているか確認
3. なければ追加
```

#### 問題3: プロライン連携エラー

```
原因: LINE Official Accountの認証が必要

対処:
1. プロラインでLINE連携を再実行
2. LINE Official Account Manager で認証
```
