# 第1部：【戦略編】なぜ「マンガ・マニュアル」が最強のソリューションなのか？

## 1-1. 残酷な現実：Z世代の脳内OSと思考停止
**ユニコ**: まず、この残酷なデータを見てちょうだい。
大手コンサルティングファームの調査によると、入社3年目以内の社員の **68%** が、「社内マニュアルを一度も開いたことがない」または「開いたが1ページで閉じた」と回答しているの。

**ウルフ**: 笑えるな。企業は何百万円もかけてマニュアルを作ってるのに、それが「デジタルゴミ」になってるってことか。

**ユニコ**: そう。理由は単純。「文字を読むのが苦痛」だからよ。
TikTok、Instagram Reels、YouTube Shorts... 彼らは生まれた時から「動画」と「画像」で情報を摂取してきた。彼らの脳内OSは、**「テキスト＝遅い、ダルい、古い」** と認識するように配線されてしまっているの。

**バナナ**: 脳科学的にも証明されています。人間の脳が画像情報を処理する速度は、テキスト情報の **60,000倍** です。
テキストのマニュアルを読ませるというのは、「光ファイバーが通っているのに、あえてモールス信号で通信しろ」と言っているようなものです。

**ダヴィンチ**: つまり、マニュアルを「マンガ」や「ビジュアル」に変換することは、単なる装飾ではない。**「情報の転送速度を6万倍にする」** という、極めて合理的な業務効率化施策なんだ。

## 1-2. NanoBanana理論：「感情」という名の記憶定着剤
**バナナ**: NanoBananaの核心をお話ししましょう。
人間は「情報」だけでは動きません。「感情」が動いた時に、初めて行動が変わります。

従来のマニュアル：
> 「経費精算は月末までに必ず行ってください。遅れると処理されません。」
> (感情: 無 / 反応: ふーん、後でいいや)

NanoBananaマンガマニュアル：
> **1コマ目**: 新人マナブ君が、月末に経費精算を忘れて青ざめている（汗ダラダラ）。
> **2コマ目**: 経理の鬼軍曹・サトウさんが「マナブくーん...」と背後に立つ（恐怖演出）。
> **3コマ目**: マナブ君、「二度と忘れません！」と誓う。
> (感情: 恐怖、共感 / 反応: ヤバい、すぐやらなきゃ)

**ユニコ**: これよ。**「エピソード記憶」** として脳に刻み込むの。
ただのルール（意味記憶）はすぐ忘れるけど、失敗談やストーリー（エピソード記憶）は忘れない。
マンガ化とは、**「業務ルールをエピソード記憶に変換する装置」** なのよ。

# 第2部：【技術編】NanoBanana Manga Prompts 徹底解剖

AIでマンガを作る際、最大の壁となるのが「キャラクターの顔が変わる（不整合）」問題です。
これを解決するための、具体的なプロンプトエンジニアリング技術を公開します。

## 2-1. キャラクター固定の奥義：「Character Sheet Anchoring」

**バナナ**: 多くの人は、毎回プロンプトに「黒髪の男性」と書くだけです。これでは毎回違う「黒髪の男性」が出てきます。
DALL-E 3 や Midjourney で一貫性を保つには、**「キャラクターシート（設計図）」** を定義し、それをアンカー（錨）として使います。

### 🔧 Step 1: キャラクター定義プロンプト
まず、AIに「このキャラを覚えろ」と指示します。

```markdown
# Character Definition: MANABU (The Rookie)
- **Role**: Junior Employee (22 years old)
- **Appearance**: Short, messy black hair. Round face. Large expressive eyes. Slight stubble.
- **Outfit**: Ill-fitting navy blue business suit (jacket slightly too big), white shirt, blue striped tie.
- **Personality Visuals**: Often sweating (nervous), enthusiastic smile, slouched posture when tired.
- **Art Style**: Japanese commercial manga style, black and white line art, screentones, clean ink lines, cel shading feel.
```

### 🔧 Step 2: シーン生成プロンプト（実用テンプレート）
次に、各コマを生成する際、この定義を必ず **冒頭** に置きます。

> **Prompt Template for DALL-E 3:**
>
> "Generate a manga panel in Japanese commercial black-and-white style.
> **[Character Reference]**: A young male rookie named MANABU (short messy black hair, ill-fitting navy suit, expressive face).
> **[Action]**: Manabu is holding a stack of receipts, looking utterly confused and overwhelmed. Question marks are floating around his head.
> **[Camera]**: Medium shot, front view.
> **[Background]**: A busy modern office desk with a computer monitor.
> **[Style]**: Clean line art, screentones, high contrast, professional manga quality."

**バナナ**: ポイントは、`[Character Reference]` の部分を **一言一句変えずに**毎回コピペすることです。AIは、同じ単語の並びに対して近い特徴を出そうとする習性があります。

## 2-2. 感情表現マトリクス (Expression Matrix)
**バナナ**: マンガにおいて「顔芸」は命です。以下の修飾語（Modifiers）を使い分けてください。

| 感情 | プロンプト修飾語 (Modifiers) | 効果 |
| :--- | :--- | :--- |
| **困惑** | `confused expression`, `sweat drops on forehead`, `swirly eyes`, `question marks floating` | 「わからない」状況を一目で伝える |
| **発見** | `eureka moment`, `lightbulb appearing above head`, `sparkling eyes`, `finger pointing up` | 解決策が見つかった時の高揚感 |
| **焦り** | `panicked face`, `motion blur`, `speed lines background`, `mouth wide open` | 締切間際の緊急性を演出 |
| **成功** | `victory pose`, `fist pump`, `background flowers blooming`, `super deformed (chibi) happy face` | 業務完了の達成感 |
| **上司** | `menacing shadow`, `glowing glasses`, `arms crossed`, `stern look` | 守るべきルールの厳格さ |

## 2-3. コマ割りと構図の魔術 (Panel Composition)
**テック**: HTML上でマンガを見せる場合、**「4コマ漫画形式（縦スクロール）」** がスマホに最適です。
複雑なページ構成にするより、`<img>` タグを縦に並べるだけのシンプルな構造のほうが、読むリズムが生まれます。

*   **Panel 1 (導入)**: 広角 (Wide Shot)。状況設定。
    *   Prompt: `Wide shot of the entire office floor, bustling atmosphere.`
*   **Panel 2 (展開)**: バストアップ (Medium Shot)。トラブル発生。
    *   Prompt: `Medium shot of Manabu looking at the error message on screen.`
*   **Panel 3 (解決)**: クローズアップ (Close Up)。先輩のアドバイスや解決の鍵。
    *   Prompt: `Extreme close up of the 'Save' button being pressed by a finger.`
*   **Panel 4 (結末)**: 全身 (Full Shot)。解決後の安堵。
    *   Prompt: `Full body shot of Manabu bowing to the senior, atmosphere is relaxed.`

# 第3部：【実装編】HTML-First & Chart.js 実装ガイド

**テック**: ここからは、プログラミング（Python/HTML/JS）の話です。
なぜ「HTML-First」なのか？ それは **「動的な表現」** と **「修正の容易さ」** が段違いだからです。

## 3-1. Chart.js による「動くグラフ」の実装
静止画のグラフではなく、JavaScriptで描画されるグラフを使います。

### 📊 コーディング例 (Python to HTML/JS)
`generator.py` で生成したJSONデータを、以下のHTMLテンプレートに流し込みます。

```html
<!-- visual.html template snippet -->
<div class="chart-container" style="position: relative; height:40vh; width:80vw">
    <canvas id="myChart"></canvas>
</div>

<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script>
const ctx = document.getElementById('myChart').getContext('2d');
const myChart = new Chart(ctx, {
    type: 'bar', // bar, line, pie, radar...
    data: {
        labels: {{ labels | tojson }}, // Pythonからリストを渡す ['1月', '2月'...]
        datasets: [{
            label: '部署別残業時間',
            data: {{ data_values | tojson }}, // [40, 35, 20...]
            backgroundColor: [
                'rgba(255, 99, 132, 0.2)', // 赤（危険）
                'rgba(54, 162, 235, 0.2)', // 青（安全）
                'rgba(255, 206, 86, 0.2)'
            ],
            borderColor: [
                'rgba(255, 99, 132, 1)',
                'rgba(54, 162, 235, 1)',
                'rgba(255, 206, 86, 1)'
            ],
            borderWidth: 1
        }]
    },
    options: {
        responsive: true,
        plugins: {
            title: {
                display: true,
                text: '残業時間の推移（30時間を超えると赤色）'
            }
        },
        scales: {
            y: { beginAtZero: true }
        }
    }
});
</script>
```

**ダヴィンチ**: このコードのポイントは、`backgroundColor` をデータの値によって動的に変えられる点だ。
「30時間を超えたら赤色にする」といったロジックを入れれば、**「どこが問題か」** が一瞬で伝わる。これはExcelで作った静止画グラフにはできない芸当だ。

## 3-2. HTMLからPowerPointへの変換戦略 (The Dual Export)
**テック**: クライアントは「パワポでも欲しい」と言います。HTMLで作ったリッチな表現を、どうやって劣化させずにパワポにするか？

`python-pptx` を使いますが、全てのレイアウトを再現するのは不可能です。割り切りが必要です。

1.  **テキスト**: HTMLの `h1`, `p`, `li` を抽出して、パワポのプレースホルダーに流し込む。
2.  **画像（マンガ）**: HTML内の `src` URLを取得し、`slide.shapes.add_picture()` で貼り付ける。
3.  **グラフ**: ここが難所です。Chart.jsはCanvas描画なので、そのままでは画像になりません。
    *   **解決策**: Selenium等で一度スクリーンショットを撮り、それを画像としてパワポに貼るのが現実解です。
    *   または、データそのものをPowerPointのチャートオブジェクト (`ChartData`) として再構築します（難易度高）。

我々のツール v3.0 では、**「画像として貼り付け」** を採用し、安定性を優先しました。

# 第6部：【LP制作編】「売れる」を科学するランディングページ技術

マニュアルを作ったら、次はそれを「売る」あるいは「読ませる」ためのLP（ランディングページ）が必要です。
v4.0 で実装された `LP Mode` の裏側にあるロジックを解説します。

## 6-1. 最強の構成「PASモデル」 (The PAS Framework)
**ウルフ**: 世の中のLPの9割はゴミだ。「弊社のマニュアルは素晴らしいです」と自慢しかしていない。
顧客は「自分の痛み」にしか興味がない。だから **PASモデル** しか勝たん。

1.  **P (Problem / 問題提起)**:
    *   「新人教育に時間がかかっていませんか？」
    *   顧客の抱えている痛みを明確に言語化する。これで「あ、私のことだ」と思わせる。
2.  **A (Agitation / 煽り・共感)**:
    *   「このままでは、ベテラン社員が教育に忙殺され、本来の業務が止まってしまいます。最悪の場合、共倒れになります。」
    *   傷口に塩を塗るフェーズ。問題の深刻さを突きつけ、感情を揺さぶる。
3.  **S (Solution / 解決策)**:
    *   「そこで、AIマンガマニュアルの出番です。新人が勝手に育つ仕組みを提供します。」
    *   ここで初めて商品を出す。PとAで「欲しい！」状態になっているので、スッと刺さる。

**ユニコ**: v4.0のAIは、入力された業務メモから自動的にこのPASを抽出するようにプロンプトが調整されているわ。
人間が書くとどうしても「売り込み」になるけど、AIなら冷徹に「痛み」を分析できるのよ。

## 6-2. HTML-First LPの技術的優位性
**テック**: WixやWordPressで作ると「重い」「移行が面倒」という問題があります。
我々の `lp.html` は、以下の利点があります。

*   **Single File Component**: 画像もCSSも（Base64などで）1ファイルに完結させれば、メール添付でLPを送ることも可能。
*   **爆速表示**: 余計なJavaScriptがないため、Lighthouseスコアで確実に100点が出る。これはSEO（検索順位）にも有利。
*   **レスポンシブ**: 最初からスマホ最適化（Mobile-First CSS）されている。

# 第7部：【サムネイル編】クリック率を支配する「0.1秒」の魔術

YouTubeや社内ポータルで、コンテンツが見られるかどうかは「サムネイル」で9割決まります。
v4.1 で実装された **Generate Catchy Thumbnail** の極意です。

## 7-1. 「YouTuberスタイル」の視覚文法
**バナナ**: 今の時代、綺麗なだけのサムネイルはスルーされます。必要なのは **「違和感」** と **「インパクト」** です。

1.  **極太ゴシック体 (Impact Font)**: 明朝体は使うな。線が細いと視認性が下がる。「ヒラギノ角ゴ StdN W8」あるいは「Impact」一択。
2.  **袋文字 (Text Border)**: 文字の周りを白や黒で縁取る。背景が何色でも文字が読めるようにする必須テクニック。
3.  **斜体 (Italic)**: 文字を少し傾けるだけで「疾走感」「緊急性」が出る。

## 7-2. CSSだけで作る「袋文字」のハック
**テック**: Photoshopを使わずに、HTML/CSSだけであの「YouTuber風文字」を再現するコードがこれです。

```css
.youtube-text {
    font-size: 100px;
    font-weight: 900;
    color: #fff; /* 白文字 */
    /* text-shadowを多重にかけることで強力な縁取りを作る */
    text-shadow: 
        6px 6px 0 #000, 
        -2px -2px 0 #000,  
        2px -2px 0 #000,
        -2px 2px 0 #000,
        2px 2px 0 #000;
    transform: rotate(-2deg); /* ちょっと傾ける */
}
```

この技術により、Photoshopを持っていない事務員でも、文字を入力するだけで「プロ級のサムネイル」が生成できるのです。

## 7-3. 色彩心理学による「テーマ」選定
**ダヴィンチ**: 色は言葉より早く脳に届く。コンテンツの内容に合わせて色を変えろ。

*   **⚠️ Warning Red**: 「閲覧注意」「禁止事項」「失敗事例」。警告色でクリックさせる。
    *   Gradient: `linear-gradient(45deg, #ff0000, #330000)`
*   **💠 Trust Blue**: 「公式マニュアル」「社長メッセージ」「成功事例」。信頼と知的さを演出。
    *   Gradient: `linear-gradient(135deg, #00c6ff, #0072ff)`
*   **👑 Premium Gold**: 「保存版」「完全攻略」「有料級」。価値の高さをアピール。
    *   Gradient: `linear-gradient(to right, #bf953f, #fcf6ba, #aa771c)`

AIはテキスト内容から「感情分析」を行い、これらの色を自動選択しています。
