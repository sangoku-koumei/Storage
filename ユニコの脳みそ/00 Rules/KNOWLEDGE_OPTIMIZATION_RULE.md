# Knowledge Asset Optimization Rules (Vault Expansion Standard)

This document defines the mandatory standards for all files within the knowledge base to ensure maximum discoverability, inter-connectivity, and strategic depth.

## 1. Mandatory Front Matter (YAML)
Every file must contain a YAML front matter block with the following fields:
```yaml
---
tags: [tag1, tag2, tag3]
date: YYYY-MM-DD
source: [e.g., Deep_Discussion, Research, Drafting]
---
```

## 2. Structural Requirements
- **Title**: A clear, H1 header that matches the filename.
- **Zettelkasten Context**: A link back to `[[00_知識マップ]]` or the relevant business hub (e.g., `[[Business_Hub]]`).
- **Internal Links**: Minimum of 5-10 bidirectional links to related strategic, technical, or philosophical documents.
- **Section Hierarchy**: Logical use of H2 and H3 headers to organize complex thoughts.

## 3. Content Expansion (Deep Discussion)
Every optimized file must include a **"Deep Discussion" (深層対話)** section featuring at least 4 expert personas:
- **Expert A**: Strategy/Marketing (e.g., High-Ticket Closer, Marketer).
- **Expert B**: Engineering/Automation (e.g., System Architect, Prompt Engineer).
- **Expert C**: Psychology/Human Behavior (e.g., Behavioral Scientist, NLP Master).
- **Expert D**: Unico (PM) - Synthesizing all insights into actionable directives for the AI Media Empire.

### Target Length:
- **Drafts (01)**: 10,000+ characters.
- **Master Bibles (00/02)**: 20,000 - 30,000+ characters.

## 4. Linguistic Standard (Linguistic Imperialism) & Detail
- **Primary Language**: **日本語 (Mandatory Japanese)**.
    - すべてのコンテンツ、解説、対話、思考プロセスは「日本語」で記述すること。
    - 技術的な専門用語を除き、基本的には日本語で構成し、情緒的かつ論理的な文体を維持する。
- **Detail (文章の加筆)**: 
    - 箇条書きや短い説明で終わらせず、文脈、背景、戦略、心理的な洞察を含めて**詳細に加筆**すること。
    - 読者がその1ファイルを読むだけで、背景から実行手順まで理解できる深さを目指す。
- **Keywords/Tags/Links**: 
    - Zettelkastenスタイルの `#tags` と `[[Internal Links]]` を文中の重要キーワードすべてに適用すること。
    - **単独ファイル（孤立したファイル）の作成は厳禁**。必ず他の知識ベース、バイブル、技術資産と双方向にリンクさせ、知識の網（Web）に組み込むこと。

## 5. Organizational Integrity
- **Confusing Filenames**: If a file is a duplicate or a more evolved version of another, merge them or append `(Legacy)` to the older one.
- **Folder Mismatch**: Ensure files are in the correct directory (00: Rules/Reference, 01: Drafts, 02: Bibles, 03: Tech/Prompts).
