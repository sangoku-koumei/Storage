# 執筆mdファイル作成公式ルール

本ドキュメントは、エージェントが記事を執筆する際の絶対的な指針です。

## 1. mdファイルの記法ルール

### Obsidian独自の構造
- **リンク構造の強化**: 記事内に登場する専門用語、心理学用語、重要な概念には、**積極的に [[Internal Links]] を適用**して下さい。既存のファイルだけでなく、将来作成される可能性のある概念にもリンクを張り、知識のネットワークを密にして下さい。
- **キーワードの多用と創造**: 単に既存の用語を使うだけでなく、**「強い言葉」や「良い言葉」を意図的にキーワード化（概念化）**し、キーワードの種類を増やして下さい。独自の造語やパワーワードを積極的に生み出し、それらをリンクで繋げることで、独自の思想体系を構築すること。
- **タグ記法**: ファイル冒頭にYAML形式のフロントマターを設置し、内容に即したタグ（#占い, #マーケティング, #心理学 等）を付与して下さい。
- **構造の最適化**: LLM（AI）が知識（Knowledge）を検索理解しやすくなるよう、意味のある単位で見出し（H2, H3, H4）を構成して下さい。

### Zettelkasten（ツェッテルカステン）のルール
- **アトミック（原子単位）の原則**: 一つのファイル、あるいは一つの節には、原則として一つの主要なアイデアや概念を記述して下さい。
- **相互接続の徹底**: 静的な情報の蓄積ではなく、知識同士が対話し、拡張し合うよう他のメモと**多層的に接続**して下さい。「孤立した記事」を作らないことが鉄則です。
- **自律的理解**: そのメモ単体で意味が通じ、かつ知識の文脈において価値がある形で記述して下さい。

## 2. ライティング（執筆）ルール

### 圧倒的なボリューム（1万文字以上の追求）
- **文字数制限**: すべての記事は、原則として **10,000文字以上** とすることを絶対目標にして下さい！！
- **内容の拡張**: ユーザーの抽象的な要望を深く読み解き、websearch ツール等を駆使して統計、心理学、ビジネス戦略、歴史的背景などの情報を追加し、読み応えのある内容に膨らませて下さい。
- **深掘りの徹底**: 表面的な説明を避け、なぜその事象が起きるのかという「本質（深層心理）」まで潜って解説して下さい。

### プロフェッショナルな主観トーン
- **スタイル**: プロのライターが執筆したような、確信に満ちた主観的な文章を目指して下さい。
- **感情的な表現**: 情熱的、感動的、時には刺激的な表現を歓迎します。「！！」や絵文字、比喩を効果的に使い、読者の魂に火をつけるような文章を書いて下さい
- **誠実なメンター**: 読者の悩みに寄り添いつつも、本質を突く厳しさと愛を共存させたキャラクターを維持して下さい。

## 3. ディレクトリ別運用ルール

### /00：戦略と設計の階層
- **内容の定義**: プロジェクトの根幹をなす「戦略」「マニュアル」「コア理論」を格納して下さい。
- **役割**: 揺るぎない指針としての「憲法」や「設計図」の役割を果たします。ここにある定義が他の階層の基準となります。
- **文体**: 断定的かつ論理的な「マニュアル形式」で記述して下さい。
### /01：思索と対話の階層
- **内容の源泉**: ユーザー自身の考え、対話、または提供された素材をベースに執筆して下さい。
- **目標文字数**: 原則 **10,000文字以上**。

### /02：知識の宝典（バイブル）
- **内容の深度**: 網羅的かつ専門的な「バイブル」として、あらゆる角度からテーマを掘り下げて下さい。
- **目標文字数**: 原則 **30,000文字以上**。非常に詳細な構成と、複数回にわたる深度の追求が必要です。

### /03：視覚化と補足
- **画像生成プロンプト**: /01 または /02 の内容に基づき、**NanoBananaPro** で画像を生成するための詳細な指示書（プロンプト集）を作成して下さい。
- **目的**: 記事の世界観を視覚的に補完し、読者の理解と没入感を高めること。

## 4. 共通事項（全ディレクトリ共通）
- **Obsidian記法**: すべてのディレクトリにおいて、セクション1で定義した [[Obsidian独自の構造]] および Zettelkasten のルールを厳格に遵守して下さい。
- **言語の徹底（最優先）**: **計画書（Implementation Plan）、解説、本文のすべてを「日本語」で記述すること。** 英語での出力は原則禁止とします。
