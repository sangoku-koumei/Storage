## Obsidianメモ活用ガイド

- **基本コンセプト**  
  - Markdownベースで双方向リンクを張れる知識管理ツール。  
  - `vault`(保管庫)単位でファイルを管理し、データはすべてローカル保存。  

- **ファイル作成と命名**  
  - ファイル拡張子は`.md`で統一。  
  - 日本語タイトルでもOKだが、リンクしやすい短い名前を推奨。  
  - 日付ノートは`YYYY-MM-DD`形式にすると検索しやすい。  

- **フォルダ構成の注意**  
  - 「参考資料」「進行中」「完了」などカテゴリ別に整理。  
  - 添付ファイル用フォルダを決め、`Settings > Files & Links > Default location for new attachments`で指定。  

- **リンクとタグの使い分け**  
  - `[[ノート名]]`：明示的な関連ノートへのリンク。  
  - `#タグ名`：テーマ横断で検索・フィルタ用に付与。タグは少数の主要キーワードに絞る。  
  - リンク時は候補ウィンドウから既存ノートを選択し、重複作成を防ぐ。  

- **テンプレート活用**  
  - `Templates`プラグインを有効化し、冒頭に目次やプロパティを自動挿入。  
  - 定型会議メモ、日報、読書メモなど用途別テンプレを用意。  

- **日付・タスク管理**  
  - `- [ ] タスク`形式でチェックリストを作成。  
  - `Tasks`プラグインを導入すると日付フィルタや検索が強化される。  
  - `Periodic Notes`で日次・週次ノートを自動生成。  

- **可視化機能**  
  - グラフビュー：ノート間のリンク構造を視覚化。ノート整理の偏りを確認できる。  
  - コードブロック` ```dataview `でDataviewクエリを使うとメタデータやタスクを一覧表示可能。  

- **同期とバックアップ**  
  - Obsidian Syncまたはクラウドストレージ(例: Dropbox)でマルチデバイス利用。  
  - Git連携でバージョン管理すると変更履歴を追跡しやすい。  

- **ショートカット例**  
  - `Ctrl/Cmd + O`：ノートクイック切り替え  
  - `Ctrl/Cmd + P`：コマンドパレット  
  - `Ctrl/Cmd + Shift + F`：全文検索  

- **セキュリティ注意**  
  - パスワードや個人情報は暗号化ノートや別ツールで管理。  
  - プラグインは信頼できる作者のものを使用し、設定で自動更新をこまめに確認。  

- **推奨ワークフロー**  
  - 収集：思いついたアイデアは即ノート追加。  
  - 整理：リンクを張り、タグ付けで関連付け。  
  - 振り返り：グラフビューやDataviewで定期レビュー。

  ## 便利ワザと具体的な使い方

- **クイックキャプチャ収集箱**  
  - `Inbox`ノートを1枚作り、`Ctrl/Cmd + Shift + I`で一時メモを追記。  
  - 1日の終わりに`Inbox` → 各プロジェクトへリンクor移動。

- **Zettelkasten風メモ**  
  - ノートIDを`20251119-アイデア名`のように付けてユニーク化。  
  - 関連ノートへ必ずリンク2本以上を設定し、孤立ノードを防止。

- **読み物ハイライト整理**  
  - 原文引用は`> 引用文`、気づきは箇条書きと`[[関連ノート]]`で接続。  
  - Dataviewで「読了日」「評価」プロパティを持つ読書ログを一覧化。

- **会議ログ + タスク連動**  
  - 会議メモテンプレ例：冒頭に`Participants`/`Agenda`/`Decisions`/`Next Actions`。  
  - `- [ ] ToDo #next`形式で書き、`Tasks`プラグインで担当者や期限をクエリ。

- **週間レビューの自動化**  
  - `Periodic Notes`で週次ノートを生成し、Dataviewで`status = "進行中"`なプロジェクトを集約表示。  
  - レビュー済みなら`status = "完了"`に変更し、次週ノートへリンク。

- **プロジェクトダッシュボード**  
  - 1プロジェクト＝1トップノート。概要、ゴール、進捗タスク、参考資料リンクを1ページに。  
  - ````dataview`で`tag=project/alpha`のタスクのみ抽出し、状況が一目で分かるように。

- **デイリーノートを指揮塔に**  
  - 朝：今日の3つの重要タスクを記入。  
  - 日中：メモや参考リンクを貼り、晩に`#win`や`#lesson`タグで振り返り。  
  - 翌日ノートへ`[[2025-11-20]] ← ToDo引き継ぎ`を書き残しておく。

- **リンク化で検索不要**  
  - メールやチケット番号など識別子をノート名にしておき、`[[TCK-1234]]`で瞬時に参照。  
  - バックリンクペインで「どこから参照されているか」を確認し、情報源を遡れる。

- **モバイル速記とデスクトップ整理**  
  - スマホで素早く箇条書き→帰宅後デスクトップでタグ付け＆リンク化。  
  - Obsidian Syncや外部クラウド利用時は衝突防止のため編集デバイスは1台ずつに。

- **安全な外部共有**  
  - `Publish`を使わない場合でも`MarkdownをPDFに変換→共有`で誤編集を防止。  
  - `git`でブランチを分け、共有前に差分レビューをして機密流出を回避。


  ## 便利な使い方追記

- **画像ドラッグ＆ドロップ**  
  - 画像ファイルをノートへ直接ドロップすると`![[ファイル名.png]]`が自動挿入され、添付フォルダへ保存される。`Settings > Files & Links > Default location for new attachments`で保存先を統一。  

- **クリップボード貼り付け**  
  - `Win + Shift + S`でスクリーンショット後、Obsidianに戻って`Ctrl + V`すると即画像化。ファイル名は日付や内容でリネームして検索性を確保。  

- **外部画像のMarkdown記法**  
  - Web上の画像は通常Markdownで`![説明文](https://example.com/image.png)`。ローカルに保存できない場合に便利だが、リンク切れ対策で重要画像はローカル保存を推奨。  

- **画像に説明を添える**  
  - 画像直後に箇条書きで要点を言語化し、`[[関連ノート]]`へリンク。Dataviewで`type: image`などプロパティ管理するとギャラリー化が容易。  

- **ExcalidrawやCanvasで図示**  
  - `Excalidraw`プラグインで手書き図を作成し、Obsidian内に保存して`![[図.excalidraw]]`で埋め込み。`Canvas`を使えば画像・ノート・リンクを同一ボードで俯瞰できる。  

- **画像の再利用**  
  - 既存画像を再利用する際は`![[添付フォルダ/diagram.png|400]]`のようにパイプで表示幅を指定。複数ノートから参照しても実体は1つのため、容量を抑えられる。  

- **PDFやスキャンの扱い**  
  - PDFの特定ページを参照したい場合、`![[document.pdf#page=3]]`と追記すると3ページ目が埋め込み表示される。紙資料の写真もまとめて添付し、メモで要約。  

- **画像付きテンプレート**  
  - 会議メモなどで図版スペースを確保したいときはテンプレートに`## 図解`セクションとダミー画像リンクを用意。差し替え漏れが発生せず整理しやすい。


  ### 任意で使える便利機能
- `[[リンク]]`・`#タグ`・日付ノート・Dataview等は強制しない
- ただし、情報の再利用性や検索性が向上すると判断した場合は積極的に活用
- 使った機能・理由が他メンバーにも分かるよう、必要に応じてノート本文に補足を書く
